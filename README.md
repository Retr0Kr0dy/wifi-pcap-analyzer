After looking at many wardriving and Wi-Fi pentesting projects, I noticed that a surprising number of them try to run the cracking engine directly on the ESP. That led to the idea of a tiny, single-file HTML sleuthing kit for 802.11 PCAPs, with a simple web-based cracker that can be served from the ESP while the actual cracking runs on the client device instead of the ESP.
